Set详情分析

Written by ruichunjie with ♥ on April 10, 2019 in 编程 · 技术

Set

      用于存储无序(存入和取出的顺序不一定相同)元素，值不能重复。

HashSet

      线程不安全，存取速度快。底层是以hash表实现的。哈希表存放的是哈希值。HashSet不存入重复元素的规则，使用hashcode() 和equals()
      当试图把对象加入HashSet时，HashSet会使用对象的HashCode来判断对象加入的位置。同时也会与其他已经加入的对象的hashCode进行比较。哈希值一样会继续比较equals()方法，如果结果为true，HashSet就视为同一个元素，否则不是同一元素。
      add()返回的是boolean类型。
      HashSet和ArrayList集合中都有判断元素是否相同的方法(boolean. contains(Object o))。HashSet使用了hashCode()和equals()方法，ArrayList使用了equal()方法

TreeSet

      红-黑树的数据结构，默认对元素进行自然排序（String）。如果在比较的时候两个对象返回值为0，那么元素重复。
      红黑树的规则: 左小右大
      比较时有两种方式:1>元素自身具备比较性，实现Comparable接口；2>容器具有比较性，实现Comparator接口。当两种形式都存在时，以comparator的比较方式为主。比较时通过return 0来判断唯一性，所以不能直接return 0

LinkedHashSet

      会保存插入的顺序。
